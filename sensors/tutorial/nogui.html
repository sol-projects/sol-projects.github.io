Building manually

Requirements:
- Linux or MacOS
- CMake version 3.18 - 3.20
- CXX 20 compliant Compiler
- Optional Requirements:
- an internet connection
- Git

To manually build the project, build the github project with the commands -DBUILD_GUI=OFF and -DBUILD_TESTS=OFF if you don't want tests.
Example: 
cmake  -DBUILD_GUI=OFF -DBUILD_TESTS=OFF -S . -B Debug
cmake --build Debug
cd Debug
./ProjectName

Getting load/temperature of a component at an interval

Use the flag -t to get the temperature and -l to get load of a component. Specify the type of the component with -d and the interval (ms) at which new data is displayed into the console with -i. The below code outputs the temperature of the CPU into the console every 100 ms. NOTE: you can only get temperature or load (not both) from exactly one device using this method.
app -t -d=CPU -i=100

Getting load/temperature of a component and saving it to a file

Use the flag -f to specify you want the information to be saved in a CSV file instead of the console. The code below saves the temperature and load of CPU and GPU into 2 seperate files at an interval of 150 ms, one file for each device. The file is named yyyy_mm_dd_hh_mm_ss_interval_component, where the first part indicates start time of the measurements. To watch all devices, we would use -d=all
app -tlf -d=CPU,GPU -i=150

Setting a future measurement

Use the flags -s for the measurement start date and -e to set the end date. The format is the ISO 8601 standard of YYYY-MM-DDTHH:MM:SS.
app -tlf -s="2023-1-18T19:13:00" -e="2023-1-18T20:00:00" -d=CPU,GPU -i=150
